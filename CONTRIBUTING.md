## Branches

feat/<description>
ex feat/add_new_links

## Commits

fix: <commit_message>
ex fix : fix button isses


## PRs

should match commits and branching style and should include a quick summary what was achieved. 
examples of what to add to summaries include why the PR was made, how to test it, etc.

## Commiting to main

Commit and merge to a dev branch and once merge conflicts are fixed and dev is stable merge dev to main.
